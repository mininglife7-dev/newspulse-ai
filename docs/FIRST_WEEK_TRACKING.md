## Key Metrics Reference

### Health Thresholds

| Metric | Target | Warning | Critical |
|--------|--------|---------|----------|
| Error rate | <1% | 1-5% | >5% |
| Uptime | >99% | 95-99% | <95% |
| Response p95 | <500ms | 500-1000ms | >1000ms |
| Database connections | <50 | 50-150 | >150 |
| Support tickets/day | <5 | 5-10 | >10 |
| Signup success rate | >95% | 90-95% | <90% |

### Locations of Key Dashboards

- **Vercel dashboard:** https://vercel.com/dashboard
  - Look for: Analytics, Function Logs, Environment Variables
- **Supabase dashboard:** https://supabase.com/dashboard
  - Look for: Database → Query Performance, Auth → Users
- **Email delivery:** Check your email provider's dashboard for bounce rates
- **Support email:** Monitor mininglife7@gmail.com for customer issues

---

## Escalation Triggers

**Immediately notify Founder if:**
1. Error rate exceeds 10% for 30 minutes
2. Uptime drops below 95% in any 24-hour period
3. Security incident detected
4. 5+ support tickets mentioning the same issue
5. Database connection pool near capacity (>150/200)
6. Customer data loss or corruption detected

---

## Post-Launch Debrief (After Day 7)

Schedule a retrospective meeting to discuss:
1. Actual vs. expected signup rate
2. Top customer issues and how they were resolved
3. Performance under load (any unexpected bottlenecks?)
4. Team feedback (what was hard? what was smooth?)
5. Product learnings from customer feedback
6. Recommended improvements for Week 2

Document findings in `docs/INCIDENTS/YYYY-MM-DD-week1-retrospective.md`
