# DNA-219: Founder Action Board Protocol

**Status:** Active — effective 2026-07-09. Companion to DNA-218 (two-layer reporting): DNA-218 governs the report body; DNA-219 governs how every mission report concludes.

Every mission report shall conclude with a standardized Founder Action Board.

## Section 1 — Mission Status

- 🏛 Mission Status
- Overall: GO / NO-GO
- Engineering: PASS / WARNING / FAIL
- Business: PASS / WARNING / FAIL
- Launch: XX%
- Confidence: XX%

## Section 2 — Founder Action Board

Only list actions that require Founder authority. Do not include engineering tasks that can be completed autonomously.

Format:

- 🔴 **Immediate (today)** — Estimated time / Estimated impact / Blocking / Decision required
- 🟡 **This Week** — Estimated time / Estimated impact
- 🟢 **Later** — Optional improvements

## Section 3 — Autonomous Engineering Queue

Display work already underway without requesting approval. For each item show: Status · Owner · Progress · Risk · ETA.

The Founder should never need to ask: "What is engineering doing now?"

## Section 4 — Launch Countdown

Display: Days Remaining · Critical Path · Completed Gates · Remaining Gates · Highest Risk · Next Verification.

## Section 5 — Mission Integrity

State explicitly: Verified Facts · Measured Evidence · Open Assumptions · Founder Decisions · Tool Limitations.

Differentiate project limitations from tooling limitations.

## Section 6 — Final Recommendation

End every report with exactly one sentence:

> "If I were optimizing purely for launch readiness, the single highest-return Founder action today would be: __________."

Only recommend one action. Never provide multiple competing priorities.
