# ADR 0002 — Static Content-Security-Policy (nonce CSP rejected on evidence)

**Date:** 2026-07-10 · **Status:** Accepted

## Context

The app served no `Content-Security-Policy`. A strict, nonce-based CSP
(`script-src 'nonce-…' 'strict-dynamic'`) is the gold standard, and Next.js
documents a middleware pattern for it.

## What was tried

The documented nonce pattern was fully implemented: middleware generated a
per-request nonce, forwarded it via the `Content-Security-Policy` request
header, and set the response header. The smoke suite then failed with
evidence:

> `response HTML does not carry the CSP nonce`

Root cause: `/` and `/history` are **statically prerendered** (`○ Static`
in the build output). Static HTML is generated once at build time and
cannot carry per-request nonces, so the browser would block every inline
bootstrap script — breaking hydration on exactly the pages users hit
first. Nonce CSP in the app router requires forcing dynamic rendering on
every page.

## Options

1. **Force dynamic rendering everywhere + nonce CSP.** Strongest policy,
   but converts every page view into a server render, losing static/CDN
   serving for a small app whose pages are otherwise cacheable. Rejected:
   the runtime cost is paid by every visitor to mitigate a vector (inline
   script injection) that the app's current surface doesn't expose — React
   escapes all rendered data and nothing renders user-controlled HTML.
2. **Static CSP without nonces** _(chosen)_:
   `default-src 'self'; script-src 'self' 'unsafe-inline'; …;
frame-ancestors 'none'; object-src 'none'; base-uri 'self';
form-action 'self'; connect-src 'self'`.
   Keeps static prerendering. Still blocks: loading any external script,
   clickjacking via framing, plugin content, `<base>` hijacking, form
   exfiltration to foreign origins, and any fetch/XHR to non-self origins.
   Concedes: inline script execution (`'unsafe-inline'`), the XSS
   defense-in-depth layer nonces would provide.
3. **Hash-based CSP for static pages.** Next.js does not emit stable
   hashes for its bootstrap scripts across builds; unmaintainable.

## Consequences

- CSP + companion headers (`X-Content-Type-Options`, `X-Frame-Options`,
  `Referrer-Policy`, `Permissions-Policy`) are served on every response
  from `next.config.js` and smoke-tested.
- An e2e test drives the app in a real browser and asserts zero CSP
  violation console messages — the policy is verified not to break
  hydration or API calls.
- Revisit nonce CSP if the app ever renders user-controlled HTML or moves
  to dynamic rendering for other reasons.
