## Operational Status Criteria

Do not claim a mission is "fully integrated" or "fully operational" until supported by execution evidence.

A mission is considered **operationally proven** only after ALL of the following are demonstrated:

### EURO AI — Operational Proof Requirements

1. **Successful end-to-end customer onboarding** — First customer progresses from registration through workspace creation without blocker
2. **Successful first customer outcome** — Customer completes first meaningful workflow (compliance assessment, risk analysis, or equivalent) with measurable result
3. **Production validation** — Customer actions execute against production infrastructure; system state transitions are correct
4. **Verified customer evidence** — Governor records customer progression with timestamps, action records, system responses, and dual verdicts (Technical + Customer-Success)
5. **Customer-critical risks resolved or accepted** — All blocking issues (BLOCKER classification) either resolved or explicitly accepted by Founder with documented business rationale

**Current EURO AI Status:** IN PROGRESS (Step 1 complete; Steps 2–14 await RISK-009 resolution + customer submission)

### VAJRA — Operational Proof Requirements

1. **Local repository discovered and verified** — VAJRA codebase location identified; branches and Git history accessible; build/test/backtest systems catalogued
2. **Scientific baseline established** — Current state of VAJRA documented (code, tests, research artifacts, outstanding questions)
3. **One complete scientific improvement cycle executed:**
   - Observe (system behavior or research question identified)
   - Hypothesize (predicted improvement or test outcome)
   - Experiment (code change, test, or analysis executed)
   - Measure (quantitative results recorded)
   - Verify (results reproducible and independent)
   - Learn (pattern captured for reuse)
   - Recommend next experiment (evidence-backed next steps)

   All steps supported by recorded evidence (code commits, test output, measurement logs, analysis documents).

**Current VAJRA Status:** DISCOVERED (repository location confirmed; awaiting Phase 0 discovery script output for baseline establishment)

## Reporting Discipline

Executive reports shall distinguish between:

| Status                            | Definition                                                              | Example                                                                                |
| --------------------------------- | ----------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| **Proven by Evidence**            | Demonstrated through execution with recorded results                    | "1342 tests pass; customer registration succeeded (run 29599513287)"                   |
| **In Progress**                   | Currently executing; partial results exist                              | "Phase 2 Step 1 complete; Steps 2–14 in progress"                                      |
| **Waiting on External Authority** | Blocked by Founder decision or customer action; Governor cannot proceed | "Phase 2 blocked on RISK-009 (email config decision)"                                  |
| **Planned**                       | Designed; resources prepared; awaiting unblocking                       | "VAJRA Phase 1+ adapter design ready; awaiting discovery output"                       |
| **Unknown**                       | Status unverified; assumed but not confirmed                            | "VAJRA repository location — assumed C: drive; requires discovery script confirmation" |

**Do not promote future capability to current capability.**

**Evidence always takes precedence over aspiration.**

### Example Discipline

**Incorrect:** "EURO AI is operationally ready."  
**Correct:** "EURO AI infrastructure is production-healthy (1342 tests pass, deployment ready). Customer operational proof requires completion of Phase 2 Steps 2–14 with documented customer success. Currently blocked on RISK-009 (email config) and customer Step 2 submission."

**Incorrect:** "VAJRA integration is designed."  
**Correct:** "VAJRA integration adapter framework designed. VAJRA Phase 0 discovery planned. Actual integration requires VAJRA repository discovery (awaiting Windows script execution) and baseline establishment."
