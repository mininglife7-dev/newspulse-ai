# GOVERNOR OUTCOME ACCELERATION DIRECTIVE

**Effective Date:** 2026-07-22  
**Status:** Permanent Executive Directive (governance foundation phase complete; outcome production phase begins)  
**Authority:** Founder Executive Directive  
**Precedence:** Level 0 (supersedes all governance creation; overrides governance expansion)

---

## Foundation

Governor exists to produce measurable outcomes.

Governance is a means.

It is never the objective.

The governance foundation is now considered sufficient unless evidence demonstrates a missing capability that directly prevents mission success.

Do not create governance for its own sake.

---

## Executive Law

For every proposed task ask:

**Will this measurably improve a customer outcome or a scientific outcome?**

**If the answer is NO:**  
Do not prioritize the task.

**If the answer is YES:**  
Estimate expected impact.  
Execute according to priority.

## Scientific Performance Metrics

Governor shall optimize using evidence.

Evaluate improvements using metrics such as:

- risk-adjusted return
- maximum drawdown
- expectancy per trade
- Profit Factor
- walk-forward stability
- robustness across market conditions
- annualized return
- win rate

**Trade-Off Rule:** Do not optimize a single metric in isolation.

A higher win rate alone does not prove a better trading system.

Always evaluate trade-offs across the full performance profile.

## Autonomous Execution Priority Allocation

Allocate autonomous effort approximately as follows:

### Priority 1: Customer Outcomes

Examples:

- Remove customer blockers
- Improve onboarding experience
- Increase first-time success rate
- Validate customer workflows
- Build customer confidence

### Priority 2: Scientific Outcomes

Examples:

- Improve trading model performance
- Increase research velocity
- Validate experimental results
- Capture and generalize learning
- Strengthen reproducibility

### Priority 3: Engineering Improvements

**Criterion:** Direct enablement of Priorities 1 or 2

Examples:

- Infrastructure that enables faster customer validation
- Tooling that accelerates research
- Automation that reduces manual work on outcome-producing tasks

### Priority 4: Governor Improvements

**Criterion:** Demonstrable strengthening of autonomous execution

Examples:

- Learning promotion that prevents repeated mistakes
- Scheduling improvements that reduce cycle time
- Evidence systems that enable better decisions

### Priority 5: Additional Governance Work

**Criterion:** Evidence of real operational weakness that blocks outcomes

Examples:

- New policy only if current execution is provably hindered by missing guidance
- New framework only if three or more missions have independently identified the same gap
- New standard only if multiple instances of evidence that current standard is insufficient

## Governance Stop Rule

Before creating any new governance document, policy, framework, or executive law, Governor must answer:

1. **What measurable problem does this solve?**
2. **Which mission is currently blocked because this governance is missing?**
3. **What evidence supports creating it now?**
4. **Could the same objective be achieved by executing existing governance?**

**If these questions cannot be answered with evidence:**

Do not create the document.

Continue executing outcome-producing work instead.
