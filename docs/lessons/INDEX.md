## By Role

### I'm a Governor Planning a New Stage

1. Read lessons from previous stages (STAGE_1/2/3_LESSONS.md)
2. Review decision retrospectives (DECISION_RETROSPECTIVES.md)
3. Check incident retrospectives (INCIDENT_RETROSPECTIVES.md)
4. Apply patterns observed in LEARNING_LOG.md

### I'm a Founder Reviewing Institutional Health

1. Check latest entries in LEARNING_LOG.md
2. Review DECISION_RETROSPECTIVES.md quarterly
3. Read stage lessons when each stage completes
4. Use patterns to inform decisions

### I'm a New Team Member

1. Read all stage lessons (STAGE_1/2/3_LESSONS.md)
2. Skim LEARNING_LOG.md to understand culture
3. Check DECISION_RETROSPECTIVES.md to understand values
4. Use this to understand "how we think"

### I'm Closing a Phase/Project

1. Document "What Went Well" (specific accomplishments)
2. Document "What Was Harder" (surprises, underestimates)
3. Document "What We'd Do Differently" (improvements)
4. Update LEARNING_LOG.md with summary
5. Share with team and Governor Ω

### I'm Responding to an Incident

1. After resolution, complete incident retrospective
2. Add key lessons to INCIDENT_RETROSPECTIVES.md
3. Update LEARNING_LOG.md with patterns observed
4. Share lessons at next team sync

## Learning Format

Every lesson should answer these questions:

```
What Was the Situation?
[What happened]

What Was the Outcome?
[Positive or negative result]

Why Did This Happen?
[Root cause or contributing factors]

What Did We Learn?
[The teachable moment — applicable principle]

How Do We Apply It?
[Concrete next steps or pattern to follow]

Impact
[What changes as a result of this lesson]
```

---

## Key Learning Themes

Watch for these patterns across lessons:

### Process Improvements

- If something took longer than expected, how do we streamline it?
- If something worked well, how do we repeat it?

### Technical Insights

- Patterns that work across multiple domains
- Common mistakes and how to prevent them
- Technology choices that proved wise or problematic

### Governance & Authority

- Decision protocols that worked
- Communication patterns that improved clarity
- Authority boundaries that need clarification

### Team & Culture

- How collaboration improved
- What practices strengthened team coherence
- Skills that need development
