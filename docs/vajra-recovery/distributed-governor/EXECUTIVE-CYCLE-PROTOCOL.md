# Executive Cycle Protocol

**Mission:** OMEGA — "From Scientist to Executive."
**Judged by:** the quality of decisions enabled, **not** the amount of work
performed. *Every execution cycle must leave the organization wiser than
yesterday.*

This is the operating discipline every Governor cycle runs. It is deliberately
short: an executive instrument, not a report.

## The value test (gate every cycle)

A cycle produces Founder value only if it improves **at least one** of:

1. a **capability**
2. a **scientific instrument**
3. a **bottleneck** (removed or better understood)
4. a **decision** (or the process that makes decisions)
5. a **learning process**
6. a **protected scientific asset**

If a cycle improves none of these, it produced little value — **say so, and
stop**, rather than manufacturing activity. Building against an unreleased
constraint has diminishing returns; a real executive surfaces the constraint
instead of padding output.

## The cycle (Distributed Governor Loop)

```
Windows: Observe → Verify → Publish Evidence
Cloud:   Analyse → Learn → Prioritize → Recommend Mission
Windows: Execute (only if Founder-authorized) → Observe Again
```

In default READ-ONLY mode the cycle runs to **Recommend Mission** and stops at
the authorization gate. Recommendations accumulate; nothing executes without a
scoped Founder token.

## Cycle close — the permanent product (Learning Law)

Every cycle appends one entry to `CYCLE-LEDGER.jsonl` answering:

- What **surprised** us? Which **assumption failed**?
- Which **capability/instrument improved**?
- Which **evidence strengthened**? Which **weakened**?
- What should **never** be repeated? What should **always** be repeated?
- Which of the six value-targets did this cycle improve? (If none → flagged.)
- **Confidence delta** — and, critically, whether *execution/scientific*
  confidence moved. It may not, and that must be stated honestly (no data ⇒ no
  movement).

## The Founder's Daily Review (answer every cycle)

1. Did we improve the organization?
2. Did we improve observation?
3. Did we remove a bottleneck?
4. Did we preserve a scientific asset?
5. Did we eliminate a false belief?
6. Did we increase confidence?
7. **What is the highest-value mission tomorrow?**

Answered in the Founder report and recorded in the ledger.

## The gates that bound ambition

- **Physics Law.** Before searching for alpha, determine whether the market
  *permits* alpha. Never optimize inside an economically impossible region.
- **Execution Law.** Do not estimate spread/slippage/latency before *measuring*
  them. Assumptions are replaced by observations, never the reverse.
- **Research Law (Alpha Campaign 2 gate).** No new strategy search until all
  five preconditions in `research-gate.json` are met. The gate is the single
  canonical instrument for this decision (supersedes prose gates elsewhere).
- **Capability Law.** Health = produced-correct-on-time **output**, not uptime,
  not heartbeat, not green logs.

## Scientific laws (non-negotiable)

Evidence before Assumption · Reality before Simulation · Observation before
Optimization · Economic Viability before Statistical Significance · Capability
before Activity · Learning before Repetition · Root Cause before Repair ·
Unknown remains UNKNOWN until measured.

## What "success" is (and is not)

Not more reports, code, hypotheses, or dashboards. Success is **better
decisions, evidence, engineering, learning, execution, scientific integrity,
and capital protection.** Protect reality. Protect evidence. Protect learning.
Protect capital.
